# ADR 0001 — Record architecture decisions

## Status

Accepted — template only.

## Context

The Savings Tracker codebase uses feature-first modules and Expo Router. Some choices will change as the product grows.

## Decision

Significant structural or cross-cutting decisions **must** be captured as ADRs in `docs/adr/` using the next sequential number.

## Consequences

- Small refactors inside one feature do **not** require an ADR.
- ADRs are short (roughly one screen) and focus on trade-offs, not implementation trivia.
